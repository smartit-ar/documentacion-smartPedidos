Resumen General de la Versión

Esta actualización ajusta la programación de tareas automáticas (Cron Jobs) para optimizar los tiempos de ejecución y evitar colisiones horarias. Además, refina la lógica de conciliación de rechazos para PedidosYa, corrigiendo un problema crítico de identificación de órdenes en la base de datos.

Análisis Técnico Detallado

api/src/controllers/branch.js

Análisis General

Se modifican los horarios de ejecución de tareas para distribuir mejor la carga vespertina y se corrige un bug en la búsqueda de órdenes a rechazar.

Cambios Detallados

Re-programación de Horarios Cron (Peya y Rappi)

Explicación: Se adelantan los horarios de PedidosYa en 1 minuto (de :02 a :01) y se añaden nuevas ejecuciones a las 17:00hs tanto para Peya como para Rappi, ampliando la cobertura de procesamiento en hora pico.

Código Anterior: 

// ANTES: Ejecuciones en minutos :02 y :32
  const pedidosYaTimes = [
    { hour: 23, minute: 2 },
    { hour: 10, minute: 2 },
    // ...
  ];
  const rappiTimes = [
    { hour: 11, minute: 16 },
  ];
Código Nuevo: 

// DESPUÉS: Ejecuciones adelantadas a :01 y :31, más nuevos slots de las 17hs
  const pedidosYaTimes = [
    { hour: 23, minute: 1 },
    { hour: 10, minute: 1 },
    // ...
    { hour: 17, minute: 1 }, // Nuevo horario
  ];
  const rappiTimes = [
    { hour: 11, minute: 16 },
    { hour: 17, minute: 16 }, // Nuevo horario
  ];
Corrección de Identificador en rejectOrdersPeya

Explicación: Se cambia el criterio de búsqueda de order.id a originalId con conversión explícita a String. Esto asegura que el ID externo de rechazo se mapee correctamente con el registro interno.

Código Anterior: 

// ANTES: Búsqueda por 'order.id', posiblemente incorrecto para este contexto
          let fullOrder = await orderModel.findOne({
            'order.id': reject.orderId
          });
Código Nuevo: 

// DESPUÉS: Búsqueda por 'originalId' con cast a String para seguridad de tipos
          let fullOrder = await orderModel.findOne({
            'originalId': String(reject.orderId)
          });

Conclusión del análisis

Optimización Operativa: Distribución estratégica de la carga de procesos en segundo plano.
Integridad de Datos: Corrección crítica (Bug Fix) que asegura que los rechazos automáticos de PedidosYa localicen y actualicen la orden correcta.
