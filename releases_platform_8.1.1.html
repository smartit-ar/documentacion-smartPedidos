Notas de Release: Platform 8.1.1

Análisis de Cambios en la Integración con MercadoPago

Resumen General

Este documento detalla tres commits consecutivos que introducen mejoras significativas en la integración con MercadoPago, abordando desde la gestión de rechazos y la corrección de bugs críticos hasta la mejora de la observabilidad del sistema.

Commit: f476e1ad

Análisis Funcional

Este cambio enriquece drásticamente la capacidad del sistema para gestionar los rechazos de pedidos de MercadoPago. Antes, la lista de motivos estaba vacía, obligando a usar razones genéricas. Ahora, se introduce una lista detallada y estandarizada que permite mejorar la comunicación con el cliente, habilitar un análisis de datos más profundo y sentar las bases para futuras automatizaciones.

Commit: b6cd1395

Análisis Funcional

Este es un bug fix crítico. El sistema fallaba al asociar los pedidos de MercadoPago con la sucursal correcta porque usaba un identificador incorrecto (brandId). Esto probablemente causaba que todos los pedidos de esta plataforma fallaran. La solución usa el campo correcto (id), desbloqueando el flujo de procesamiento.
