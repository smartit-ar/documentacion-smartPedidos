Flujo de Obtención de Tokens de MercadoPago

Análisis General

El siguiente documento describe el proceso de autenticación de múltiples pasos para obtener y desencriptar un token de acceso de MercadoPago. Este flujo centralizado involucra al servicio SmartFran Cloud para mejorar la seguridad y la gestión de credenciales.

Pasos del Flujo de Autenticación

Paso 1: Obtener el Token de Autenticación de la API Cloud

Primero, es necesario generar un token de autenticación (tokenAPICloud) para autorizar las solicitudes posteriores a la API de SmartFran Cloud.

Método: POST

Endpoints:

  • Test: https://smartfran-cloud-platform-dev.azurewebsites.net/api/v1/Security/token
  • Prod: https://smartfran-cloud.us.auth0.com/oauth/token

Cuerpo (Body) de la solicitud:

Test: 

{
    "client_id": "xxx",
    "client_secret": "xxx",
    "audience": "https://smartfran-cloud-catalog.azurewebsites.net/"
}
Prod: 

{
    "client_id": "xxx",
    "client_secret": "xxx",
    "audience": "smartfran-cloud-prod",
    "grant_type": "client_credentials"
}
Paso 2: Obtener el Token Cifrado de MercadoPago

A continuación, se utiliza el tokenAPICloud obtenido en el paso anterior para solicitar el token de acceso de MercadoPago, el cual se recibirá en formato cifrado.

Método: GET

Endpoints:

  • Test: https://smartfran-cloud-business-test.azurewebsites.net/api/v1/IntegratedEntity/GetAccessTokenMPByUser/{user_id}
  • Prod: https://smartfran-cloud-business-pro.azurewebsites.net/api/v1/IntegratedEntity/GetAccessTokenMPByUser/{user_id}

Encabezados (Headers):


Authorization: Bearer {tokenAPICloud}
TraceKey: 1234
TenantId: d3186bc6d7b2
Paso 3: Desencriptar el Token de MercadoPago

El token devuelto en el Paso 2 está codificado en Base64 y debe ser desencriptado para poder utilizarse.

  • Algoritmo de Cifrado: AES-128-CBC
  • Key: xxx
  • IV (Vector de Inicialización): xxx

Nota: Este proceso es automatizado para la obtención de tokens en el Platform.

Datos Importantes a Disponer (Test y Prod)

Para que el flujo completo funcione, es necesario contar con las siguientes credenciales para cada entorno:

  • "client_id"
  • "client_secret"
  • Key
  • IV
