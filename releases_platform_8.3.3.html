Análisis General

Este commit corrige lógica de negocio de Mercado Pago para evitar posibles errores en la comunicacion con su API.

Los cambios clave incluyen:

  • Logs: Se añade más contexto a los registros de error.
  • Validaciones: Se mejora la forma en que se comprueban las respuestas de Mercado Pago.
  • Correcciones Técnicas: Se ajusta la sintaxis y se añade claridad al código con comentarios.

Cambios Detallados

1. Logs

Explicación: Se ha mejorado el registro de errores en varios puntos para incluir el objeto completo de la order, además, en un caso, se eliminó información redundante para hacer el log más conciso.

Código Anterior: 

// ANTES: Varios logs de error no incluían el objeto 'order', dificultando saber qué pedido falló.
// ...
if (!mercadoPago) {
  Log.saveError(null, { message: 'MercadoPago platform not found in DB', platform: 'MercadoPago' });
  resolve(false);
}
// ...
try {
  tokenMercadoPagoCloud = await axios.get(urlTokenMercadoPagoCloud, headersToken);
} catch (err) {
  Log.saveError(null, { message: 'Error fetching token from Cloud',urlTokenMercadoPagoCloud: urlTokenMercadoPagoCloud,headersToken:headersToken , details: err.response?.data || err.message, platform: 'MercadoPago' });
  resolve(false);
}
Código Nuevo: 

// DESPUÉS: Ahora se incluye 'data: order' en los logs, proporcionando contexto completo.
// ...
if (!mercadoPago) {
  Log.saveError(null, { message: 'MercadoPago platform not found in DB', data: order, platform: 'MercadoPago' });
  resolve(false);
}
// ...
try {
  tokenMercadoPagoCloud = await axios.get(urlTokenMercadoPagoCloud, headersToken);
} catch (err) {
  // DESPUÉS: Se simplifica el log, eliminando la URL y los headers que ya no se consideran necesarios para el diagnóstico.
  Log.saveError(null, { message: 'Error fetching token from Cloud', details: err.response?.data || err.message, platform: 'MercadoPago' });
  resolve(false);
}
2. Validación de Token

Explicación: La comprobación del token obtenido desde el Cloud ahora es más estricta. Antes, solo se verificaba que la respuesta tokenMercadoPagoCloud no fuera nula. Ahora, se utiliza "optional chaining" (?.) para verificar de forma segura que la respuesta contenga la estructura esperada (data.token).

Código Anterior: 

// ANTES: La validación era débil. Una respuesta exitosa pero sin token (ej. { data: {} }) pasaría el filtro.
if (!tokenMercadoPagoCloud) {
  Log.saveError(null, { message: 'Failed to get MercadoPago token from Cloud', platform: 'MercadoPago' });
  resolve(false);
}
Código Nuevo: 

// DESPUÉS: La validación es más segura y específica. Solo continúa si 'tokenMercadoPagoCloud.data.token' existe.
if (!tokenMercadoPagoCloud?.data?.token) {
  Log.saveError(null, { message: 'Failed to get MercadoPago token from Cloud', data: order, platform: 'MercadoPago' });
  resolve(false);
}
3. Claridad del Código y Correcciones Menores

Explicación: Se han añadido comentarios para clarificar los pasos importantes del flujo. Además, se corrigió la estructura del objeto headersConfig para la llamada a la API de MercadoPago, anidando correctamente el objeto de cabeceras y añadiendo el Content-Type.

Código Anterior: 

// ANTES: La estructura de headers era incorrecta para axios y faltaba el Content-Type.
const headersConfig = {
  'Authorization': `Bearer ${decryptedTokenMercadoPagoCloud}`
}
Código Nuevo: 

// DESPUÉS: La estructura es la correcta y se añade el Content-Type.
const headersConfig = {
  headers: {
    'Authorization': `Bearer ${decryptedTokenMercadoPagoCloud}`,
    'Content-Type': 'application/json',
  }
};

Conclusión del análisis

Este commit es un refinamiento técnico que no altera la lógica de negocio principal, pero sí mejora significativamente la calidad del código.

  • Mayor Observabilidad: Los logs enriquecidos facilitarán la identificación y resolución de problemas en producción.
  • Mayor Robustez: Las validaciones más estrictas y el uso de "optional chaining" hacen que el código sea más resistente a respuestas inesperadas de servicios externos.
  • Mejores Prácticas: Las correcciones en la estructura de las cabeceras y la adición de comentarios mejoran la legibilidad y mantenibilidad del código.
