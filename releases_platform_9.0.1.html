Resumen General

Este release contiene modificaiones puntuales en la integración con la plataforma **Rappi**. Los cambios se enfocan en proceso de confirmación de pedidos . Se garantiza que el tiempo de preparación enviado a Rappi sea siempre un número entero y se mejora el log de errores con más detalles.

Análisis Técnico Detallado

api/src/platforms/management/platform/rappi.js

Análisis General

Este commit introduce dos pequeñas pero importantes mejoras en la clase `Rappi`. La primera es una corrección técnica para asegurar que el tiempo de preparación enviado a la API de Rappi sea siempre un número entero. La segunda es una mejora en los logs.

Cambios Detallados

1. Conversión a Entero del Tiempo de Preparación

Explicación: Se ha añadido una línea para convertir explícitamente el tiempo de preparación (`timeMins`) a un entero usando `parseInt()`. Anteriormente, el cálculo del promedio `(time.maxMinutes + time.minMinutes) / 2` podía resultar en un número decimal (ej. 22.5). La API de Rappi espera un número entero en la URL, y este cambio asegura que siempre se envíe un valor en el formato correcto, evitando posibles errores de validación por parte de la API externa.

Código Anterior: 

// ANTES: El cálculo del promedio podía generar un número decimal.
// ...
                let time = deliveryTimes.find(r => r.id.toString() === preparationTime);

                let timeMins = (time.maxMinutes + time.minMinutes )/2;
                 /* LOGIN  */
                const xAuth = this.tokenRappi[order.country];
// ...
Código Nuevo: 

// DESPUÉS: Se asegura que 'timeMins' sea siempre un número entero antes de usarlo en la URL.
// ...
                let time = deliveryTimes.find(r => r.id.toString() === preparationTime);

                let timeMins = (time.maxMinutes + time.minMinutes )/2;
                timeMins = parseInt(timeMins);
                 /* LOGIN  */
                const xAuth = this.tokenRappi[order.country];
// ...
2. Mejora en el Log de Errores

Explicación: Se ha mejorado el log que se registra cuando falla la petición de confirmación a la API de Rappi. Ahora, además del mensaje de error, se incluyen la `url` a la que se intentó llamar y las `options` (que contienen los headers de la petición).

Conclusión del análisis

Este commit contiene dos modificaciones: La conversión a entero del tiempo de preparación previene potenciales errores de `400 Bad Request` por parte de la API de Rappi si esta no acepta valores decimales en la URL. Y el log de error al proporcionar toda la información de la petición fallida en un solo lugar.
