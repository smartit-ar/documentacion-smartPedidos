Notas de Release: Platform 8.4.1

Actualización de la URL de autenticación de Cloud en el entorno de producción.

Información de Despliegue

Versión: 8.4.1

Fecha de Despliegue: 2025-09-09

Entorno: Producción

AWS TASK: 171

Resumen General

Este release se centra en una actualización de la configuración para el entorno de producción. Se ha modificado la URL de autenticación del servicio Cloud.

Análisis Técnico Detallado

api/src/config/env/production.js

Análisis General

Este commit realiza una actualización en la configuración del entorno de producción. Se modifica la URL del endpoint de autenticación del servicio "Cloud". El cambio es para el correcto funcionamiento de la autenticación en el entorno productivo, ya que apunta a una Url provista por el CLoud en vez de una de 0Auth.

Cambios Detallados

1. Actualización de la URL de Autenticación

Explicación: Se ha modificado el valor de la propiedad `url` dentro del objeto `cloudParams`. La URL anterior apuntaba a un servicio de Auth0, mientras que la nueva URL apunta a un endpoint específico (`/api/v1/Security/token`) en un servicio alojado en Azure (`smartfran-cloud-platform-pro.azurewebsites.net`).

Código Anterior: 

// ANTES: La autenticación se realizaba contra un endpoint de Auth0.
// ...
module.exports = {
  // ...
      cloudParams: {
        client_id: 'xxxx',
        client_secret: 'xxxx',
        audience: 'xxx',
        url: 'https://smartfran-cloud.us.auth0.com/oauth/token',
        api: 'xxx',
        grant_type:'xxx'
      },
// ...
Código Nuevo: 

// DESPUÉS: La autenticación ahora se realiza contra un nuevo endpoint en la plataforma de Azure.
// ...
module.exports = {
  // ...
      cloudParams: {
        client_id: 'xxxx',
        client_secret: 'xxxx',
        audience: 'xxx',
        url: 'https://smartfran-cloud-platform-pro.azurewebsites.net/api/v1/Security/token',
        api: 'xxx',
        grant_type:'xxx'
      },
// ...

Conclusión del análisis

Se ha modificado el valor de la propiedad `url` dentro del objeto `cloudParams`. La URL anterior apuntaba a un servicio de Auth0, mientras que la nueva URL apunta a un endpoint específico (`/api/v1/Security/token`) en un servicio alojado en Azure (`smartfran-cloud-platform-pro.azurewebsites.net`).
