Resumen General de la Versión

Este release introduce una nueva funcionalidad crítica para la observabilidad del sistema. Se implementa un endpoint en el Concentrador (POST /monitoring/orderTimes) que permite consultar y filtrar las métricas de latencia de órdenes. Esta API actúa como la interfaz de lectura para los datos generados por la Plataforma, permitiendo filtrar por rangos de fecha, plataformas, sucursales y tipos de despliegue, asegurando que solo se devuelvan registros con ciclos de vida completos.

Implementación de getOrderTimes (Lógica de Filtrado)

Explicación: Se crea la función principal que maneja la normalización de fechas, la resolución asíncrona de nombres de plataforma y la validación de integridad mediante $exists: true.

Código Nuevo: 

// DESPUÉS: Nueva función completa
const getOrderTimes = async (req, res) => {
  try {
    const { selectedPlatforms, dateSelectedFrom, dateSelectedTo, comboCloud, sucursal, orden } = req.body;
    const filters = {};

    // 1. Filtro de Fechas (Obligatorio)
    if (!dateSelectedFrom || !dateSelectedTo)
      return res.status(400).json({ error: 'Fechas obligatorias' });
    
    // Normalización de timestamps
    const fromDate = new Date(dateSelectedFrom + 'T00:00:00.000Z');
    const toDate   = new Date(dateSelectedTo   + 'T23:59:59.999Z');
    filters['latencyTimestamp.entryDate'] = { $gte: fromDate, $lte: toDate };

    // 2. Filtro de Plataformas (Resolución ID -> Nombre)
    if (Array.isArray(selectedPlatforms) && selectedPlatforms.length > 0) {
      const platforms = await platformModel.find({ internalCode: { $in: selectedPlatforms } });
      filters.platform = { $in: platforms.map(p => p.name) };
    }

    // 3. Filtros Opcionales
    if (sucursal) filters.branch = sucursal;
    if (orden) filters.order = orden;
    if (comboCloud !== 'all') filters.cloud = (comboCloud === 'cloud');

    // 4. Integridad de Datos (Solo órdenes completas)
    filters['latencyTimestamp.platformDate'] = { $exists: true };
    filters['latencyTimestamp.responseDate'] = { $exists: true };
    filters['latencyTimestamp.sendSqsDate'] = { $exists: true };
    filters['latencyTimestamp.receiveSqsDate'] ={ $exists: true };
    filters['latencyTimestamp.finalResponseDate'] ={ $exists: true };

    const ordertimes = await orderTimeModel.find(filters);
    res.status(200).json(ordertimes);
  } catch (error) {
    res.status(400).json(error).end();
  }
}

Conclusión del análisis

Observabilidad: Provee una herramienta precisa para auditar el ciclo de vida de las órdenes.
Calidad de Datos: Filtra el ruido de órdenes incompletas asegurando que solo se analicen registros con traza completa.

Análisis General

Se crea el modelo de Mongoose correspondiente a la colección donde se almacenan los tiempos, compartido con el sistema Platform.

Cambios Detallados

Definición del Esquema

Explicación: Se define un esquema flexible (strict: false) para soportar la evolución de latencyTimestamp y se habilitan timestamps automáticos.

Código Nuevo: 

import mongoose, { Schema } from 'mongoose'
import uniqueValidator from 'mongoose-unique-validator'

const orderTimeSchema = new Schema({
    order: { type: String },
    branch: { type: String },
    platform: { type: String },
    latencyTimestamp: { type: Object }
}, {
    strict: false,
    timestamps: true
});

orderTimeSchema.plugin(uniqueValidator);
orderTimeSchema.plugin(require('@meanie/mongoose-to-json'));

module.exports = mongoose.model('orderTime', orderTimeSchema);
