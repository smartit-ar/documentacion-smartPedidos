Notas de Release: Platform 9.2.19

Actualización de lógica de recepción de pedidos en integraciones.

Información de Despliegue

Versión: 9.2.19

Fecha de Despliegue: 2025-11-25

AWS Task: 193

Commits: 2664fb9d94a3d4f512b8df6da69af48ee4f6429f

Resumen General de la Versión

Se implementó una estandarización en el método de recepción de pedidos (receiveOrder) para 6 plataformas de delivery (MercadoPago, PedidosYa, Rapiboy, Rappi, ThirdParty, UberEats). La actualización permite la aceptación de pedidos con estado intermedio (statusId 9), además del estado 1 previamente soportado, mejorando la compatibilidad con los flujos de las plataformas externas.

Análisis Técnico Detallado

api/src/platforms/management/platform/mercadoPago.js

Análisis General

Se actualiza la lógica condicional dentro del método receiveOrder. Anteriormente, la integración solo procesaba órdenes con statusId === 1.

Cambios Detallados

Expansión de validación de estado

Explicación: Se modifica la condición if para incluir el statusId === 9 como un estado válido para aceptar el pedido.

Código Anterior: 

if (order.statusId === 1) {
    // Lógica de procesamiento
}
Código Nuevo: 

if (order.statusId === 1 || order.statusId === 9) {
    // Lógica de procesamiento
}

Conclusión del análisis

La integración de MercadoPago ahora acepta pedidos que llegan con estado 9, evitando que sean descartados.
