Resumen General de la Versión

Esta actualización se centra en mejorar la observabilidad y el diagnóstico de problemas en la comunicación con plataformas externas. Se reemplazan los logs de texto plano por objetos estructurados ("metadata-first") que incluyen ID de orden, sucursal, país y detalles técnicos de la petición HTTP, unificando el formato para MercadoPago, PedidosYa, PediGrido, Rappi, UberEats y servicios ThirdParty.

Análisis Técnico Detallado

api/src/platforms/management/platform/mercadoPago.js

Análisis General

Se reemplazan los mensajes descriptivos genéricos por una estructura estandarizada que incluye metadatos detallados para mejorar la trazabilidad.

Cambios Detallados

Estandarización de Logs de Error

Explicación: Se implementa una estructura de logs enriquecida. Se añade un objeto order con IDs y ubicación, y un objeto request con los detalles de la comunicación fallida.

Código Antes: 

Log.saveError(error, { message: 'Falló petición confirmación orden', url: url, body: body, headersConfig: headersConfig, platform: 'MercadoPago' });
Código Después: 

console.log(error)
Log.saveError(error, {
  message: 'Platform Communication Error: MercadoPago [action] ' + order.id,
  action: '[action]', 
  platformId: 8, 
  order: {
    "id": order.id,
    "originalId": String(order.id),     
    "statusId": order.statusId,     
    "branchId": order.branchId,
    "country": currentBranch.address.country
  }, 
  request: { 
    url: url,
    body: body,
    header: headersConfig
  }
});

api/src/platforms/management/platform/pedidosYa.js

Análisis General

Se aplica la normalización masiva de logs cubriendo casi todo el flujo operativo de una orden (recepción, confirmación, despacho, rechazo).

Cambios Detallados

Normalización Masiva de Acciones

Explicación: Se implementa la misma estructura de metadatos que en MercadoPago, estandarizando el reporte de errores para todas las interacciones críticas con la API de PedidosYa.

Código Antes: 

Log.saveError(error, { message: 'Falló peticion confirm Peya' + order.id, platform: 'PedidosYa' });
Código Después: 

console.log(error)
Log.saveError(error, {
  message: 'Platform Communication Error: PedidosYa receiveOrder ' + order.id,
  action: 'receiveOrder', 
  platformId: 1, 
  order: {
    "id": order.id,
    "originalId": String(order.id),     
    "statusId": order.statusId,
    "branchId": order.branchId,
    "country": currentBranch.address.country
  }, 
  // ... request details
});

api/src/platforms/management/platform/rapiboy.js (PediGrido)

Análisis General

Además de normalizar los logs de PediGrido, se integra una gestión detallada de errores para las comunicaciones con el sistema de lealtad SML.

Cambios Detallados

Logs Estructurados y Soporte SML

Explicación: Se estandarizan los logs para el ID de plataforma 7 y se añade la categoría Sml Communication Error para las peticiones de puntos y canjes.

Código Antes: 

Log.saveError(error, { message: 'Falló peticion confirm rapiboy ' + order.id, platform: 'PediGrido' });
Código Después: 

console.log(error)
Log.saveError(error, {                     
  message: 'Platform Communication Error: PediGrido receiveOrder ' + order.id,                    
  action: 'receiveOrder', 
  platformId: 7, 
  order: {
    "id": order.id,             
    "originalId": String(order.id),     
    "statusId": order.statusId,
    "branchId": order.branchId,              
    "country": currentBranch.address.country
  }, 
  // ... request details
});

Conclusión del análisis

Metadata-First: Migración a objetos estructurados para facilitar diagnósticos.
Consistencia: Todas las plataformas reportan errores bajo el mismo formato.
Depuración: Visibilidad inmediata con console.log previos al guardado.
