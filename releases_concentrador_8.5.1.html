Resumen General de la Versión

Esta actualización se centra en la estabilización y el monitoreo del proceso automático (Cron) encargado de calcular los promedios de latencia. Se asegura la disponibilidad de dependencias críticas (SendGrid), se implementa un log de auditoría al inicio del proceso y se optimiza la captura de errores para garantizar la visibilidad de fallos.

Análisis Técnico Detallado

api/src/utils/initialData.js

Análisis General

Se mejora la robustez del script de inicialización de datos. Se resuelven dependencias implícitas y se mejora la secuencia de registro de errores para evitar pérdidas de información en caso de fallos en el servicio de correo.

Cambios Detallados

Importación Explícita de Dependencias

Explicación: Se agrega la importación de @sendgrid/mail al inicio del archivo. Anteriormente, su uso dentro del bloque catch podía causar un ReferenceError si la librería no estaba disponible en el contexto global.

Código Nuevo: 

import settings from '../config/settings';
const sgMail = require('@sendgrid/mail') // Nueva dependencia explícita
Instrumentación de Inicio de Proceso

Explicación: Se incorpora un log de inicio mediante Log.saveError con la etiqueta platform: 'Test'. Esto permite confirmar visualmente en la base de datos cuándo se dispara el cron job, facilitando el debugging.

Código Nuevo: 

export class initialData { 
  static async ordertimesAvgCron() {
    try {
      // Log de auditoría de inicio
      Log.saveError(null, { message: 'ingreso [CRON] Calculando promedio dia anterior orderTimes...', platform: 'Test' });  
      console.log('[CRON] Calculando promedio dia anterior orderTimes...');
Reordenamiento de Manejo de Errores

Explicación: Se mueve el registro del error de base de datos (Log.saveError(dbError...)) al principio del bloque catch. Esto asegura que el error original quede registrado inmediatamente, antes de intentar enviar el correo de notificación, que es una operación susceptible a fallos de red.

Código Anterior: 

    } catch (dbError) {
      sgMail.setApiKey(settings.SENDGRID_API_KEY);
      // ... lógica de envío de mail ...     
      Log.saveError(dbError, { message: 'Falló envío mail initialData', platform: 'SendGrid' }); // Al final
    }
Código Nuevo: 

    } catch (dbError) {
      // Registro inmediato del error crítico
      Log.saveError(dbError, { message: 'Falló envío mail initialData', platform: 'SendGrid' });  
      sgMail.setApiKey(settings.SENDGRID_API_KEY);
      // ... lógica de envío de mail ...
    }

Conclusión del análisis

Estabilidad: Se eliminan puntos de fallo potenciales por referencias faltantes.
Observabilidad: Se garantiza que tanto el inicio del proceso como los errores críticos sean auditados correctamente en los logs del sistema.
