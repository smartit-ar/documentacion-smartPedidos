Notas de Release: Backoffice v6.3.1

Corrección en la configuración de rutas para el control de acceso.

Información de Despliegue

Versión: 6.3.1

Fecha de Despliegue: 26/01/2026

Commits: d2913310a50cd0e3c8134d4e7ae0edf90b8e52ee

Resumen General de la Versión

Este cambio es una corrección de configuración (Bug Fix) en el sistema de gestión de roles. Se corrige el mapeo del recurso "Latencias", alineando el valor del permiso con la ruta de navegación real del frontend en lugar de la API del backend, garantizando así el acceso correcto al módulo.

Análisis Técnico Detallado

app/src/app/pages/management/users/user/user.component.ts

Análisis General

Se actualiza la definición del objeto de permiso para que coincida con la ruta de Angular esperada por el AuthGuard y el menú lateral.

Cambios Detallados

Corrección del Valor del Permiso "Latencias"

Explicación: Se reemplaza la ruta de API /api/monitoring por la ruta de página /pages/monitoring/latencias. Esto permite que el sistema de frontend valide correctamente si el usuario tiene permiso para ver esa pantalla.

Código Anterior: 

// ANTES: El permiso apuntaba a un endpoint de backend, inútil para navegación frontend
    {
      permission: "Latencias",
      value: "/api/monitoring",
      check: false,
      tabPosition: 1
    },
Código Nuevo: 

// DESPUÉS: El permiso apunta a la ruta real de la página Angular
    {
      permission: "Latencias",
      value: "/pages/monitoring/latencias",
      check: false,
      tabPosition: 1
    },

Conclusión del análisis

Restauración de Acceso: Corrige la inconsistencia que impedía a los usuarios autorizados acceder al módulo de Latencias.
Coherencia: Alinea la configuración de permisos con el enrutamiento definido en monitoring-routing.module.ts.
