Resumen General de la Versión

Esta actualización corrige la identificación del modelo logístico en las órdenes provenientes de MercadoPago. Se implementa una validación dinámica para determinar si un pedido debe ser entregado por el restaurante (Delivery Propio) basándose en la estructura de costos de la orden, reemplazando la asignación estática anterior.

Análisis Técnico Detallado

api/src/platforms/interfaces/mercadoPago.js

Análisis General

Se modifica la interpretación del objeto de la orden para detectar automáticamente el flag ownDelivery. La presencia de cargos adicionales (otherFees) es el indicador técnico utilizado para identificar que el comercio gestiona su propia flota de reparto.

Cambios Detallados

Detección Dinámica de Delivery Propio (ownDelivery)

Explicación: Se reemplaza el valor false por defecto por una evaluación condicional. Si data.total.otherFees.value es mayor a 0, el sistema marca el pedido como "Delivery Propio".

Código Anterior: 

// ANTES: Se asumía siempre que NO era delivery propio
          order.pickupDateOnShop = null;
          order.preOrder = false;
          order.observations = data?.displayId ? `#${data.displayId}` : '';
          order.ownDelivery = false;
          order.tenant = {
            name: tenant.name,
Código Nuevo: 

// DESPUÉS: Se determina si es delivery propio basándose en la existencia de 'otherFees'
          order.pickupDateOnShop = null;
          order.preOrder = false;
          order.observations = data?.displayId ? `#${data.displayId}` : '';
          order.ownDelivery = data.total?.otherFees?.value > 0 ? true :  false;
          order.tenant = {
            name: tenant.name,

Conclusión del análisis

Precisión Logística: Permite distinguir correctamente pedidos que requieren flota propia de aquellos recogidos por la plataforma, evitando confusiones operativas en el restaurante.
