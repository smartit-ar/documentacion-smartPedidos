1. Propósito y Arquitectura General

Objetivo

Implementar un sistema de reglas de negocio dinámicas (ej. tiempo de preparación, auto-aceptación de pedidos) que dependan de la combinación chain-platform. El sistema debe ser flexible, permitiendo que sucursales individuales sobrescriban las reglas definidas por su cadena para manejar excepciones.

Arquitectura

Se establece una jerarquía de reglas de dos niveles:

  • Nivel chain (Fuente de Verdad Principal): La colección chains contendrá un conjunto de reglas por defecto para cada plataforma con la que opera.
  • Nivel Sucursal (Excepciones): La colección branches podrá, de manera opcional, definir su propio conjunto de reglas para una plataforma específica, ignorando la configuración de la cadena para ese caso.

Cuando el sistema necesite una regla, seguirá el siguiente flujo lógico:

  • ¿Existe una regla específica en la sucursal? → Sí → Usar la regla de la sucursal.
  • ¿Existe una regla específica en la sucursal? → No → Usar la regla por defecto de la cadena.

2. Modelos de Datos y Estructura en MongoDB

2.1. Modelo chain Actualizado

Se añade el array platformRules al esquema chainSchema. Cada elemento de este array define un "contrato" de reglas para una plataforma específica a nivel de toda la cadena.

api/src/models/chain.js

import mongoose, { Schema } from 'mongoose';
import uniqueValidator from 'mongoose-unique-validator';

const chainSchema = new Schema({
    chain: { type: String, maxlength: 100, required: true },
    // ... otros campos existentes
    
    // --- NUEVO CAMPO PARA LAS REGLAS DE NEGOCIO POR DEFECTO ---
    platformRules: [{
        platformId: {
            type: Schema.Types.ObjectId,
            ref: 'platform',
            required: true
        },
        rules: {
            autoAccept: { type: Boolean, default: false, required: true },
            preparationTime: { type: Number, enum: [5, 10, 15, 20, 25, 30], required: true }
        }
    }]
}, {
    strict: false,
    timestamps: true
});

// Índice para asegurar que no se duplique una regla para la misma plataforma
chainSchema.index({ _id: 1, "platformRules.platformId": 1 }, { unique: true });

chainSchema.plugin(uniqueValidator);
module.exports = mongoose.model('chain', chainSchema);

2.2. Modelo branch Actualizado

Dentro del array platforms de la sucursal, se añade un campo opcional rules. La presencia de este campo indica una excepción a la regla de la cadena.

api/src/models/branch.js

import mongoose, { Schema } from 'mongoose';

const branchSchema = new Schema({
    branchId: { type: Number, unique: true, required: true },
    chain: { type: Schema.Types.ObjectId, ref: 'chain' },
    // ... otros campos existentes
    
    platforms: [{
        platform: { type: Schema.Types.ObjectId, ref: 'platform' },
        branchReference: { type: Object },
        isActive: { type: Boolean, default: true },
        // ... otros campos de plataforma

        // --- CAMPO OPCIONAL PARA SOBRESCRIBIR REGLAS ---
        // Si este objeto existe, sus valores tienen prioridad sobre los de la cadena.
        rules: {
            autoAccept: { type: Boolean },
            preparationTime: { type: Number }
        }
    }]
    // ... resto del esquema
}, { timestamps: true });

module.exports = mongoose.model('branch', branchSchema);

    "branchId": 5162,
    "chain": { "$oid": "3b0ed8fa297fb07209439589" },
    "platforms": [
        {
            "platform": { "$oid": "a24d12527267ecfd0b946390" }, // PedidosYa
            "rules": { // <-- REGLA ESPECÍFICA DE LA SUCURSAL
                "preparationTime": 25 // Sobrescribe el valor de 15 de la cadena
            }
        }
    ]
}
