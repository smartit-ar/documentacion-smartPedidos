Resumen General de la Versión

Esta actualización transforma la capacidad de observabilidad del sistema mediante la automatización. Se introduce una tarea programada (Cron Job) y un motor ETL para calcular estadísticas de latencia diariamente a las 8:00 AM. Además, se optimiza el almacenamiento de métricas mediante la creación de colecciones analíticas agregadas, reduciendo la carga en consultas de reportes históricos.

Análisis General

Actúa como orquestador de tareas. Se añade una nueva programación diaria para invocar la lógica de cálculo de promedios de manera autónoma.

Cambios Detallados

Importación y Programación de Tarea de Promedios

Explicación: Se define el schedule 0 8 * * * (diariamente a las 8 AM) para ejecutar initialData.ordertimesAvgCron(), asegurando el procesamiento de los datos del día anterior.

Código Anterior:

cron.schedule(scheduleOrderRejClosed, async () => await orderRejClosed());
Código Nuevo: 

// DESPUÉS: Se añade la tarea de cálculo de promedios
import initialData from '../utils/initialData';
// ...
const scheduleOrerTimesAvg = '0 8 * * *';  
cron.schedule(scheduleOrerTimesAvg, async () => await initialData.ordertimesAvgCron());
