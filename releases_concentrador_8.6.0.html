Resumen General de la Versión

Esta actualización representa un salto cualitativo en la automatización de la plataforma. Se integra formalmente el partner PediGrido con un control activo de su estado (online/offline) y horarios comerciales. Además, se mejora la resiliencia del sistema mediante la adopción de Promise.allSettled y la creación de modelos de datos específicos para la gestión offline-first de horarios.

Análisis Técnico Detallado

api/src/config/env/*.js

Análisis General

Se introduce la configuración necesaria para interactuar con la API de PediGrido en los distintos entornos, permitiendo la autenticación y gestión de sucursales.

Cambios Detallados

Nuevas Credenciales de Partner

Explicación: Se agrega el objeto pediGridoParams con usuario, contraseña y URL base, integrando al nuevo socio en el ecosistema de configuración.

Código Nuevo: 

  pediGridoParams: {
    usuario: '',
    password: '',
    baseUrl: '' // Configurado según entorno (staging/prod)
  },

api/src/controllers/branch.js

Análisis General

El controlador evoluciona de un sistema reactivo a uno proactivo. Se implementan tareas programadas (Crons) para la gestión automática de apertura/cierre de PediGrido y se mejora la robustez en llamadas masivas.

Cambios Detallados

Robustez con Promise.allSettled

Explicación: Se sustituye Promise.all por Promise.allSettled. Esto garantiza que el fallo en la actualización de una sucursal no interrumpa el proceso para el resto de la flota.

Código Nuevo: 

// Se utiliza Promise.allSettled para evitar interrupciones masivas
await Promise.allSettled(branchReferences.map(async (rappiNumber) => { ... }));
Automatización de PediGrido

Explicación: Se configuran nuevos Cron Jobs para login, chequeo de estado offline y sincronización de horarios, permitiendo una gestión desatendida.

Código Nuevo: 

async function cronPeya(baseUrl) {
  // ... crons anteriores ...
  cron.schedule(schedulePediGridoLogin, async () => await pediGridoLogin());
  cron.schedule(schedulePediGridoOffline, async () => await checkOfflinePOSpedigrido());
  cron.schedule(scheduleGetInfoBranchesGrido, async () => await getInfoBranchesGrido());
}

api/src/models/ (branch.js, gridoBranch.js)

Análisis General

Se actualizan los esquemas de base de datos para persistir el estado de conectividad del nuevo partner y almacenar localmente sus horarios comerciales.

Cambios Detallados

Nuevo Flag en Branch

Explicación: Se añade alreadyClosedPediGrido para controlar el estado y evitar envíos redundantes de comandos.

Código Nuevo (branch.js): 

    alreadyClosedPediGrido: {
       type: Boolean 
    },
Nuevo Modelo gridoBranch

Explicación: Se crea un modelo especializado para almacenar horarios de "takeaway" por día, base para la toma de decisiones del concentrador.

Código Nuevo (gridoBranch.js): 

const gridoBranchSchema = new Schema({
    branchId: { type: Number, required: true, unique: true },
    branchReference: { type: String, required: true },
    horarios: {
        takeaway: {
            lunes: { type: Array },
            // ... resto de la semana
        }
    },
    updatedAt: { type: Date, required: true, default: Date.now }
});

Conclusión del análisis

Resiliencia: El uso de allSettled aísla fallos puntuales.
Automatización: Los nuevos crons reducen drásticamente la operación manual.
Escalabilidad: El sistema está preparado para operar offline-first gracias al nuevo modelo de datos.
