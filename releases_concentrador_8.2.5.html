Resumen General de la Versión

Este release contiene un importante *bug fix* en el controlador de sucursales (`branch.js`) relacionado con la persistencia de datos de configuración de plataformas. La corrección asegura que el **`mercadoPagoUserId`** —un identificador crucial para la integración con MercadoPago— no se pierda accidentalmente al actualizar otras configuraciones de la sucursal. La lógica de actualización ahora preserva explícitamente este campo, mejorando la robustez y la integridad de los datos de configuración de la plataforma.

api/src/controllers/branch.js

Análisis General

Esta actualización, como parte de una **Mejora en la Persistencia de Datos de Plataforma**, consiste en un *bug fix* dentro de la función `updateOne`. El propósito del cambio es asegurar que el campo **`mercadoPagoUserId`**, un identificador crucial para la integración con MercadoPago, no se pierda accidentalmente durante una actualización general de la configuración de plataformas de una sucursal.

Anteriormente, la lógica que preservaba los datos existentes al actualizar una plataforma solo consideraba el campo `rules`. Esta modificación extiende esa lógica para incluir también el `mercadoPagoUserId`. El impacto funcional es un aumento en la robustez del sistema: se previene la pérdida de datos de configuración importantes, garantizando que la integración con MercadoPago siga funcionando correctamente después de que se realicen cambios en otras propiedades de la plataforma.

Cambios Detallados

1. Preservación Explícita de `mercadoPagoUserId` en `updateOne`

Explicación: Se ha modificado el bloque de código dentro de la función `updateOne` que se encarga de fusionar los datos de las plataformas entrantes con los existentes. La lógica `map` que recorre las plataformas ahora se asegura de que, además de preservar el campo `rules`, también se mantenga el valor de `mercadoPagoUserId` que viene en la solicitud de actualización.

Código Anterior: 

// ANTES: La lógica de preservación solo consideraba el campo 'rules'.
// ...
    if (params.platforms) {
      params.platforms = params.platforms.map(p => {
        const existing = currentBranch.platforms.find(ep => ep.platform.toString() === p.platform);
        return {
          ...p,
          rules: existing?.rules
        };
      });
    }
// ...
Código Nuevo: 

// DESPUÉS: La lógica ahora también se asegura de incluir 'mercadoPagoUserId'.
// ...
    if (params.platforms) {
      params.platforms = params.platforms.map(p => {
        const existing = currentBranch.platforms.find(ep => ep.platform.toString() === p.platform);
        return {
          ...p,
          rules: existing?.rules,
          // Se añade explícitamente 'mercadoPagoUserId' al objeto de retorno.
          mercadoPagoUserId: p.mercadoPagoUserId
        };
      });
    }
// ...

Conclusión del análisis

La modificación en `api/src/controllers/branch.js` es una corrección importante que mejora la integridad de los datos de configuración.

  • **Prevención de Pérdida de Datos:** El cambio soluciona un *bug* potencial que podría haber causado la eliminación del `mercadoPagoUserId` al actualizar otros aspectos de una plataforma, lo que habría roto la integración para esa sucursal.
  • **Robustez:** Hace que el proceso de actualización sea más robusto y predecible, garantizando que los datos de configuración críticos se conserven.
  • **Consistencia:** Asegura que la lógica de preservación de datos sea completa, incluyendo todos los campos personalizados importantes (`rules` y `mercadoPagoUserId`) que no deben ser sobrescritos accidentalmente.
