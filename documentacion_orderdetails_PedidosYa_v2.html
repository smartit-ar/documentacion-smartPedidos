Documentación de Mapeo de Órdenes: detailsMapperParseOther

Proceso de transformación de órdenes de PedidosYa al formato interno del sistema

Este documento describe la lógica utilizada para procesar y transformar los items de una orden proveniente de PedidosYa hacia el formato interno del sistema (News). El objetivo es clarificar cómo se interpretan los productos, combos/promos y sus modificaciones para analistas y usuarios.

Objetivo del Proceso

La función detailsMapperParseOther tiene como responsabilidad tomar la lista de productos brutos de una orden de PedidosYa Order y convertirlos en una estructura News, para diferenciar correctamente entre productos individuales y combos con sus respectivos componentes y personalizaciones.

Estructura de Entrada (Order)

El proceso recibe una lista de productos (products) dentro de la orden. Cada producto tiene atributos clave que determinan cómo será procesado.

Atributos del Producto

  • name: El nombre del producto. Utilizado para determinar si es un producto simple o un combo.
  • remoteCode: Código identificador (SKU). Se utiliza para validación y vinculación de componentes.
  • selectedToppings: Lista de grupos de opciones seleccionadas.
    • children: Las opciones específicas elegidas dentro de un grupo (ej: "Sin lechuga", "Extra carne", "Coca Cola").
      • remoteCode: Utilizado para la lógica de combos (vincula extras con sus ítems principales).
      • name: Utilizado para detectar eliminaciones en productos simples (ej: "Sin...").

Reglas de Negocio y Lógica de Mapeo

El sistema clasifica cada ítem de la orden en dos categorías principales basándose en su nombre: COMBO/PROMOCION o PRODUCTO.

1. Detección y Procesamiento de COMBOS

Un producto se considera un COMBO/PROMOCION si su nombre (en minúsculas) comienza con alguna de las siguientes palabras:

  • "promo"
  • "combo"
  • "hamburguesa"

Estructura Resultante para Combos

  • Tipo: COMBO/ PROMOCION
  • Precio: Se calcula sumando el precio base del combo + la suma de los precios de los componentes seleccionados ( Ej: Adicional Helado , Adicional Cerveza).

Tratamiento de Componentes del Combo (Sub-ítems)

Dentro de un combo, las opciones (children) se analizan por su remoteCode para distinguir entre un componente principal y una modificación (Extra u Opcional - "Sin" -).

  1. Componentes Principales (Items incluidos):
    • Son las opciones cuyo remoteCode NO contiene el caracter +.
    • Se añaden a la lista de includedItems del combo.
    • Su precio se suma al precio total del combo.
  2. Modificaciones (Extras y Eliminaciones):
    • Son opciones cuyo remoteCode SÍ contiene el caracter +.
    • Formato Requerido del RemoteCode: Para que el sistema sepa a qué componente pertenece el extra, el código debe seguir una lógica de vinculación (ej: contener el SKU del padre).
    • Identificación:
      • Termina en E: Se considera un Extra (agregado).
      • Termina en O: Se considera una Opcional (Sin).
    • Acción: El sistema busca el componente padre en la lista de includedItems (usando el SKU extraído del código) y le adjunta esta modificación en su respectiva lista (extras o removedComponents).

2. Detección y Procesamiento de PRODUCTOS SIMPLES

Cualquier producto que NO cumpla con los criterios de nombre de Combo se trata como PRODUCTO SIMPLE.

Estructura Resultante para Productos Simples

  • Tipo: PRODUCT
  • Precio: Precio unitario directo del producto.
  • Items Incluidos: null (no tiene sub-ítems, solo personalizaciones).

Tratamiento de Toppings/Opciones

Las opciones seleccionadas se clasifican puramente por su nombre:

  1. Componentes Opcionales (Sin):
    • Si el nombre de la opción comienza con la palabra "sin" (insensible a mayúsculas/minúsculas).
    • Se añade a la lista customizations.removedComponents.
  2. Extras/Agregados:
    • Cualquier otra opción que no empiece con "sin".
    • Se añade a la lista customizations.extras.

Ejemplo de Transformación

A continuación se muestra cómo un JSON de entrada se transforma en el formato de salida, ilustrando los casos de Combo y sus lógicas.

Entrada (Pedido de PedidosYa)

{
  "products": [
    {
      "name": "Combo - Smack + Bebida",
      "remoteCode": "20060",
      "unitPrice": "940",
      "selectedToppings": [
        {
          "name": "Smack",
          "children": [
            {
              "remoteCode": "60",
              "name": "Smack",
              "price": "0",
              "quantity": 1
            }
          ]
        },
        {
          "name": "Bebida",
          "children": [
            {
              "remoteCode": "404",
              "name": "Patagonia Amber Lager",
              "price": "100"
            }
          ]
        },
        {
            "name": "Extras",
            "children": [
              {
                "remoteCode": "17+60E", 
                "name": "Extra Carne",
                "price": "60"
              }
              // Nótese el '60' en el código, vinculándolo con la Smack (SKU 60)
            ]
        },
        {
            "name": "Quitar Ingredientes",
            "children": [
              {
                "remoteCode": "7+60O",
                "name": "Sin Bacon",
                "price": "0"
              }
              // Nótese el 'O' al final, indicando eliminación
            ]
        }
      ]
    }
  ]
}

Salida (Estructura Interna - News)

El sistema detecta "Combo" en el nombre, identifica "Smack" y "Patagonia" como componentes principales. Detecta "Extra Carne" (termina en E, linkeado al sku 60) y "Sin Bacon" (termina en O, linkeado al sku 60) y los anida dentro de "Smack".

{
  "details": [
    {
      "itemType": "COMBO",
      "itemDescription": "Combo - Smack + Bebida",
      "sku": "20060",
      "price": 1040, // 940 (base) + 100 (Patagonia)
      "includedItems": [
        {
          "itemType": "PRODUCT",
          "itemDescription": "Smack",
          "sku": "60",
          "customizations": {
            "extras": [
              {
                "itemType": "COMPONENT",
                "itemDescription": "Extra Carne",
                "price": "60"
              }
            ],
            "removedComponents": [
              {
                "itemType": "COMPONENT",
                "itemDescription": "Sin Bacon",
                "price": 0
              }
            ]
          }
        },
        {
          "itemType": "PRODUCT",
          "itemDescription": "Patagonia Amber Lager",
          "sku": "404",
          "price": "100",
          "customizations": { "extras": [], "removedComponents": [] }
        }
      ]
    }
  ]
}
