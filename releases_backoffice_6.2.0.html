Resumen General de la Versión

Esta actualización en el módulo de Dashboard (vista de Status) adapta la interfaz de usuario para soportar estructuras de datos de pedidos más complejas y heterogéneas. Los cambios responden a la necesidad de visualizar correctamente pedidos con jerarquías (combos, modificadores) y normalizar datos que provienen de diferentes integraciones con nombres de propiedades variados.

Análisis Técnico Detallado

app/src/app/pages/dashboard/status/status.component.html

Análisis General

Funcionalmente, permite que el operador vea el desglose completo de un ítem, incluyendo sus extras o componentes, lo cual antes estaba oculto o limitado. Además, hace al componente resiliente ante variaciones en la nomenclatura de los datos del backend.

Cambios Detallados

Adaptabilidad en el Renderizado de Propiedades (Fallback Logic)

Explicación: Se modifica la interpolación de datos en la tabla principal. Ahora se implementa una lógica de fallback (operador ternario) para soportar variaciones como itemDescription si description no existe, o quantity si count no existe.

Código Anterior: 



  {{ detail.sku }}
  {{ detail.description }}
  
  {{ detail.count }}
  
  
    {{
    !!detail.price
    ? selectedNew.order.payment.currency +
    detail.price * detail.count
    : "-"
    }}
Código Nuevo: 



                        
    {{ detail.sku }}
    
    {{ detail.description ? detail.description : detail.itemDescription}} 
    
    
    {{ detail.count ? detail.count : detail.quantity }} 
    
    
      {{
      !!detail.price
      ? selectedNew.order.payment.currency +
      detail.price * (detail.count ? detail.count : detail.quantity)
      : "-"
      }}
Visualización de Ítems Anidados (Included Items)

Explicación: Se introduce una sección condicional para renderizar sub-ítems (includedItems). Si un producto principal tiene componentes hijos, estos se listan debajo. Se añade lógica visual para etiquetar como "Extra" (precio > 0) o "Incluye" (precio 0).

Código Nuevo: 


  
    
      {{ item.sku }}
      
      {{  item.price > 0 ? 'Extra': 'Incluye' }}
      {{  item.itemDescription }}            
      
        {{ selectedNew.order.payment.currency  + item.price }}
      
      {{ item.quantity }}
      No
      
        {{ selectedNew.order.payment.currency  + item.price }}

Conclusión del análisis

Robustez: La interfaz asegura compatibilidad con múltiples integraciones al no fallar por nombres de propiedades distintos.
Transparencia Operativa: Los operadores ahora pueden ver qué contiene un combo o qué extras se pidieron, desglosados línea por línea.
UX: La distinción visual automática entre "Extra" e "Incluye" reduce errores en la preparación o facturación.
